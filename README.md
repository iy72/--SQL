# 基于 SQL 的消费信贷信用风险分层分析

## 一、项目简介
本项目基于多表消费信贷数据，搭建了一套 **SQL 驱动的信用风险分析流程**，用于识别高风险借款人、分析违约影响因素，并构建可解释的客户风险分层体系。

项目模拟零售金融 / 消费金融场景下的实际业务分析过程，整合了以下几类数据：
- 客户申请信息
- 历史贷款申请记录
- 分期还款明细
- 外部征信数据
- 信用卡行为数据

通过多表关联、指标汇总和规则化评分，本项目实现了从 **数据建模 → 风险分析 → 风险分层** 的完整分析闭环。

---

## 二、项目目标
本项目主要解决以下三个业务问题：

1. 哪类借款人更容易发生违约？
2. 收入水平、贷款负担、历史申请行为、逾期表现、外部负债与当前违约风险之间有何关系？
3. 能否基于 SQL 构建一套可解释的客户风险评分与分层框架？

---

## 三、数据表说明
本项目共设计 6 张核心数据表：

### 1. `dim_customer` 客户主表
存放客户基础信息、收入情况、当前贷款信息及违约标签，是项目的核心主表。

### 2. `fact_previous_application` 历史申请表
记录客户历史贷款申请情况，包括申请金额、审批结果、用途等信息，用于分析客户历史借贷行为。

### 3. `fact_installment_payment` 分期还款明细表
记录客户历史贷款的分期还款信息，用于计算逾期次数、平均逾期天数、准时还款率等指标。

### 4. `fact_bureau_credit` 外部征信表
记录客户在其他金融机构的信贷和负债信息，用于衡量客户外部信用风险。

### 5. `fact_credit_card_balance` 信用卡行为表
记录客户信用卡余额、授信额度、还款与逾期情况，用于分析信用卡使用压力及风险特征。

### 6. `agg_customer_risk_profile` 客户风险汇总宽表
基于上述多张明细表汇总形成客户级分析宽表，用于后续大部分风险分析与分层建模。

---

## 四、数据表关系
本项目的数据关系如下：

- 一个客户对应多条历史申请记录
- 一个客户对应多条分期还款记录
- 一个客户对应多条外部征信记录
- 一个客户对应多条信用卡行为记录
- 一条历史申请记录可对应多条分期还款记录

核心关联字段：
- `customer_id`：客户唯一标识
- `prev_app_id`：历史申请唯一标识

---

## 五、项目流程

### 第一步：数据建模
根据消费信贷业务逻辑，设计客户、历史申请、还款、征信、信用卡行为等关系型数据表。

### 第二步：数据清洗
完成基础数据预处理，包括：
- 缺失值检查与处理
- 主键重复检查
- 数值字段标准化
- 无效值与异常值识别
- 类别字段统一

### 第三步：客户级指标汇总
通过 SQL 多表聚合，构建客户级风险画像宽表 `agg_customer_risk_profile`，包括：
- 历史申请次数
- 历史拒贷次数与拒贷率
- 平均逾期天数、最大逾期天数
- 逾期次数与逾期率
- 外部征信负债与逾期金额
- 信用卡使用率与最大逾期天数

### 第四步：风险分析
基于 SQL 完成以下方向的分析：
- 客户画像分析
- 收入与贷款负担分析
- 历史申请行为分析
- 还款行为分析
- 外部征信风险分析

### 第五步：风险分层
基于核心风险指标构建规则化评分体系，对客户划分为：
- 低风险（Low Risk）
- 中风险（Medium Risk）
- 高风险（High Risk）

---

## 六、核心分析问题
本项目主要围绕以下问题展开分析：

1. 资产组合整体违约率是多少？
2. 不同年龄段客户的违约率有何差异？
3. 不同收入区间客户的违约风险是否不同？
4. 贷款收入比是否越高，违约率越高？
5. 历史申请次数与当前违约是否存在关联？
6. 历史拒贷率是否能反映客户风险水平？
7. 平均逾期天数与当前违约风险之间有何关系？
8. 是否发生过逾期会如何影响违约率？
9. 外部征信负债规模是否会提升违约风险？
10. 能否基于 SQL 构建规则化风险评分模型，实现客户风险分层？

---

## 七、核心指标设计
本项目重点构建了以下风险分析指标：

- `loan_to_income_ratio`：贷款收入比
- `annuity_to_income_ratio`：年金收入比
- `prev_app_count`：历史申请次数
- `prev_refused_count`：历史拒贷次数
- `prev_refusal_rate`：历史拒贷率
- `avg_days_past_due`：平均逾期天数
- `max_days_past_due`：最大逾期天数
- `late_payment_count`：逾期还款次数
- `bureau_total_debt`：外部征信总负债
- `bureau_total_overdue`：外部征信总逾期金额
- `avg_card_utilization_ratio`：平均信用卡使用率

---

## 八、主要分析结论
本项目可输出的典型结论包括：

- 贷款收入比较高的客户，违约率通常更高
- 历史申请频繁且拒贷率较高的客户，整体风险水平更高
- 存在逾期记录的客户，其当前违约率显著高于从未逾期客户
- 外部征信负债较高、存在外部逾期记录的客户，更容易集中于高风险层级
- 基于多维指标构建的规则化评分体系，能够有效区分不同风险层级客户

---

## 九、业务建议
基于分析结果，可以提出以下业务建议：

1. 对贷款收入比较高的客户提高审批审查强度
2. 对存在多次拒贷记录的客户进行重点复核
3. 对历史逾期较严重客户设置更严格的授信条件
4. 将规则化风险分层应用于贷前筛查与贷后监控
5. 对高风险客群建立重点跟踪与预警机制
